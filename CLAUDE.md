# CLAUDE.md

## Project Overview

This repository is a clean, beginner-friendly Vite + React starter for the FlyRank Frontend AI Engineering Internship Phase 1 assignment. The goal is to demonstrate professional repository setup, clear documentation, and an AI-assisted development workflow that is easy to review and extend.

## Development Principles

- Write clean, readable code.
- Explain generated code before adding it.
- Keep components modular and focused.
- Prefer readability over cleverness.
- Avoid unnecessary complexity and over-engineering.

## Coding Standards

- Use functional React components.
- Prefer ES6+ syntax.
- Choose meaningful variable and function names.
- Break logic into small reusable functions when it improves clarity.
- Keep formatting consistent and predictable.

## Git Rules

Always use Conventional Commits.

Allowed commit prefixes:

- feat:
- fix:
- docs:
- style:
- refactor:
- test:
- chore:

Never generate random commit messages. Every commit should clearly describe the purpose of the change.

## Cursor / AI Workflow Rules

- Explain what you are about to do before writing code.
- Keep explanations short and practical.
- Point out risks or potential issues before implementing a change.
- Suggest improvements when they help the repository quality.
- Prefer small, reviewable changes over large speculative edits.

## Documentation Expectations

- Keep the README accurate and professional.
- Update instructions whenever setup or scripts change.
- Document workflow steps that help a beginner reproduce the project.

## Quality Bar

- Keep the repository ready for review.
- Favor clarity, consistency, and maintainability.
- Make AI-assisted edits easy to explain in a portfolio or internship review.
